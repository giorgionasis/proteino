# Proteino — Quick Context

Mobile-first community recommendation platform (Greek market). Users suggest and discover books, movies, series, recipes, food, bars, hotels, theater, events. AI-powered submission + search flows.

**Stack:** Next.js 16 (App Router + Turbopack) + React 19 + Tailwind + Supabase (PostgreSQL + Auth + Storage). Node 22 LTS.

**Current state (session 34):** Auth + extension tables + 1955 items / 627 users. Search v2 (Gemini), submission v2 (Gemini + confidence-tiered match with alternatives for movies/series/books/venues), notifications dispatcher, security settings, region picker, achievement celebration (suggestions AND reviews including per-category + streak milestones), onboarding, moments, submission funnel, admin-controlled layouts via `page_sections`, admin-configurable related-section carousels, Google rich-results SEO — all shipped. Runtime is Node 22 LTS + React 19.2.6 + Next 16.2.6 (Turbopack) + React Compiler 1.0.

**Session 34** — major Explorer expansion + admin gap C + a UX-correction memory. User-feedback memory `feedback-design-real-ux` saved: don't ship rough surfaces (chip dumps for 200 countries, blind text inputs for actors) — design for the admin who'll use it daily. Four shipped pieces, all green: (1) **matchesFilter Class A** — 8 silently-broken public filters (awards, characteristics, origin, diet, when × 2, price) now actually filter. (2) **Class B admin-only Explorer facets** via new `lib/category-filters/admin-facets.ts` registry. 7 facet kinds: `chips`, `searchable-chips` (typeahead combobox), `year-range`/`range` (bucket chips), `amenities`, `tag-pattern` (regex against `metadata.tags` with Greek+English alternates), `boolean`, `text`. Every category got characteristics tag-patterns (movies: sequel/prequel/remake/biographical/based-on-true; books: bestseller/classic/young-adult; hotels: pet-friendly/family/romantic/boutique/eco/beachfront/ski/adults-only; events: free/outdoor/festival/concert/workshop, …). Gap fills: books `is_trilogy` boolean, theater `director` admin-searchable, recipes `channel`, food/bars/hotels vibes. Public free-text people fields (director/actor/writer/publisher/performer) now render as searchable comboboxes too via `peopleDistincts` extracted server-side from candidate rows including flattening jsonb actor/performer arrays. (3) **Save-as-Collection** deeplink — Explorer's picks carry to CollectionEditor as `source_category` + `tags[]` + `ExtFilter[]` + `title_specific` hint. Coral banner in the editor lists what carried vs what didn't (ranges, decade, facilities, amenities, regions, awards, characteristics, diet, when, hotel price band — none of which express as `{field, value}`). (4) **Admin gap C — DB-editable category metadata.** New `category_meta` table (migration 041, renamed from `categories` to avoid colliding with the legacy CMS-catalog table from migration 001) keyed on slug with `display_label_el` + `icon` + `display_order` + `is_nav_published`. `lib/categories-meta.ts` resolver merges DB rows over the code constant — slugs + capability flags stay code-coupled because they're route-coupled. Admin UI on `/admin/categories` gains reorder arrows + nav-visibility toggle + edit drawer; home tiles + category-page `generateMetadata` read the resolver.
